# Results

## Overview

Evaluation using the synthetic dataset generated by `traffic_gen`. The dataset contains 1000 packets: 850 legit and 150 malicious, interleaved at a 17:3 ratio per 20-packet block (85%/15%). The 15% malicious rate reflects a realistic low-rate attack scenario rather than an obvious flood.

The malicious set includes both structural attacks (fail early in the validator) and subtle attacks (pass basic checks and are caught by deeper behavioral rules).

---

## Dataset

| Category | Count | Description |
|---|---|---|
| Legit HTTP (port 80) | ~425 | GET, POST, PUT, HEAD requests with valid Host headers |
| Legit HTTP (port 443) | ~170 | GET and POST requests on HTTPS port |
| Legit DNS (UDP port 53) | ~255 | Valid DNS A-record queries with correct header structure |
| Malicious — HTTP on DNS port | ~18 | HTTP GET/POST payloads sent to UDP port 53 |
| Malicious — DNS on HTTP port | ~12 | Binary DNS headers sent to TCP port 80 |
| Malicious — bad HTTP method | ~12 | Unrecognized method verb (`BADVERB`) on port 80 |
| Malicious — missing Host header | ~12 | HTTP request missing required Host: field |
| Malicious — missing HTTP version | ~12 | Request line with no HTTP/1.x version token |
| Malicious — truncated DNS | ~12 | 4-byte DNS header (below 12-byte minimum) on UDP port 53 |
| Malicious — HTTP version spoof | ~12 | `HTTP/1.9` — passes method/Host, fails tightened version check |
| Malicious — DNS tunneling | ~12 | 40-char base64 subdomain label, fails label-length heuristic |
| Malicious — DNS response as query | ~12 | QR bit=1 in flags, fails QR bit check |
| Malicious — request smuggling | ~12 | Duplicate `Content-Length` header, fails duplicate-header check |
| Malicious — null byte injection | ~12 | Binary null appended after valid headers, fails null byte check |

|---|---|---|

| Accuracy ((TP + TN) / total) | 100% | — |

All 150 malicious packets were correctly flagged. All 850 legit packets were correctly cleared.

## Observations

**Protocol impersonation is reliably detected.** HTTP payloads on UDP port 53 fail immediately at the DNS validator's label-parsing step — the HTTP text doesn't form a valid DNS name. DNS payloads on TCP port 80 fail the HTTP method check since the binary DNS header doesn't start with a valid ASCII method string.

**Subtle attacks are caught by behavioral rules, not structural checks.** The new attack categories demonstrate this clearly:
- HTTP/1.9 version spoof → passes method/Host, caught at Rule 5 (tightened version: only 1.0, 1.1, 2 accepted)
- DNS tunneling (40-char label) → passes port/header/QDCOUNT, caught at Rule 6 (label > 32 bytes)
- DNS response as query (QR bit=1) → passes port/length/QDCOUNT, caught at Rule 5b (QR bit check)
- Request smuggling (duplicate Content-Length) → passes all structural rules, caught at Rule 9
- Null byte injection → passes all string-based checks, caught at Rule 8 (memchr scan)

**No false positives.** Valid HTTP GET/POST/PUT/HEAD requests with Host headers pass all 9 rules. Valid DNS queries with correct header structure and standard QTYPE pass all 7 rules.

---

## Limitations

The dataset is entirely synthetic and designed to cover the specific attack types in `traffic_gen.c`. The 100% detection rate reflects deterministic rule coverage against known patterns — it doesn't represent performance against real-world or novel traffic. Testing against real `curl` and `dig` captures would be the natural next step.
